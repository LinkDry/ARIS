# 研究想法报告：卫星网络+LLM

**生成时间**: 2026-03-22

---

## Top 5 推荐研究方向

### 🥇 Top 1: 多源日志故障根因定位 (SatFault-LLM)

**标题**: 卫星网络多源异构日志融合的故障根因智能定位

**问题**: 卫星网络故障诊断面临告警日志来源多样、级联告警多、传播路径复杂的挑战

**核心假设**:
- LLM能够理解不同厂商告警日志语义，建立统一故障表示空间
- 故障传播链路具有可学习模式，可通过LLM推理重构

**方法框架**:
1. 日志语义对齐模块
2. 告警关联图构建
3. 故障传播推理（思维链）
4. 置信度评估

**为什么LLM必需**:
- 语义理解：不同厂商告警描述差异大，关键词匹配准确率<60%
- 推理：故障传播涉及多层级，需要层次化推理
- 知识迁移：新型卫星缺乏历史数据，LLM可迁移知识

**预期贡献**:
- 故障定位准确率从65%提升至85%+
- 诊断时间从30分钟缩短至3分钟
- 构建1000+真实故障案例知识图谱

---

### 🥈 Top 2: 自然语言仿真场景生成 (SatScenario-Gen)

**标题**: 基于自然语言的卫星网络仿真场景自动生成

**问题**: 卫星网络仿真场景构建复杂，NS-3/STK学习曲线陡峭

**核心假设**:
- 自然语言描述可转换为结构化仿真配置
- 仿真场景配置具有可学习的模式和约束

**方法框架**:
1. 场景描述解析（LLM）
2. 配置参数推理
3. 仿真脚本生成（NS-3/STK）
4. 场景验证

**为什么LLM必需**:
- 自然语言理解是核心需求
- 参数推理需要领域知识
- 脚本生成需要理解仿真工具语义

**预期贡献**:
- 场景构建时间从数小时缩短至数分钟
- 降低卫星网络仿真门槛
- 构建场景描述-配置对数据集

---

### 🥉 Top 3: 运维知识问答系统 (SatOps-Assistant)

**标题**: 卫星网络运维知识助手与智能决策支持

**问题**: 运维知识分散、新员工培训周期长、故障处理依赖经验

**核心假设**:
- 运维知识可通过RAG有效组织
- LLM能理解运维问题上下文给出针对性回答

**方法框架**:
1. 多源知识融合
2. 分层知识图谱
3. RAG问答引擎
4. 操作指令生成

**为什么LLM必需**:
- 运维问题高度语义复杂
- 知识关联和推理需要语义理解
- 自然语言交互降低运维门槛

**预期贡献**:
- 问答准确率90%+
- 新员工培训周期缩短50%
- 构建卫星网络运维领域最大知识图谱
