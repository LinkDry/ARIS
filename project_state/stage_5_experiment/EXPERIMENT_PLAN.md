# 实验计划：LLM-SRAF 语义资源分配框架

## 一、实验目标

### 1.1 核心 Claims 验证

| Claim ID | 声明 | 验证实验 | 成功标准 |
|----------|------|---------|---------|
| **C1** | LLM能有效理解业务描述并提取资源需求 | Exp-1: 语义理解准确率测试 | 准确率 > 85%, F1 > 0.82 |
| **C2** | 语义感知资源分配优于传统方法 | Exp-2: 系统性能对比 | 吞吐量↑10%, 时延满足率↑15% |
| **C3** | 跨模态融合提升决策质量 | Exp-3: 消融实验 | 各模态贡献可量化 |
| **C4** | 方法具有实时性 | Exp-4: 时延测试 | 端到端时延 < 100ms |

### 1.2 实验优先级

```
优先级排序:
  P0: Exp-2 主实验 (验证核心贡献)
  P1: Exp-3 消融实验 (验证设计合理性)
  P1: Exp-1 语义理解实验 (验证基础能力)
  P2: Exp-4 实时性测试 (验证实用性)
  P2: Exp-5 Baseline对比 (验证优越性)
```

## 二、实验设置

### 2.1 仿真环境

| 组件 | 选择 | 理由 |
|------|------|------|
| **卫星网络仿真** | NS-3 + STK 或 自研Python仿真器 | 开源、可定制、支持LEO星座 |
| **卫星星座** | Starlink-like (72轨道面 × 22卫星) | 真实场景代表性 |
| **用户分布** | 全球城市分布 + 海洋热点区域 | 覆盖多样化场景 |
| **业务生成** | 泊松过程 + 业务类型混合 | 模拟真实流量 |

### 2.2 数据集构建

#### 数据集1: 业务描述-资源需求配对数据 (用于C1验证)

| 数据项 | 说明 |
|--------|------|
| **规模** | 10,000 条配对数据 |
| **来源** | 人工标注 + GPT辅助生成 |
| **业务类型** | 视频会议、视频流、IoT、文件传输、语音通话、网页浏览、游戏、直播 |
| **标注字段** | 业务类型、时延要求、带宽需求、可靠性要求、优先级 |
| **划分** | Train: 70%, Val: 15%, Test: 15% |

**样本示例**:
```json
{
  "description": "我需要进行一个紧急的国际视频会议，参与者有5人，预计持续1小时",
  "labels": {
    "service_type": "video_conference",
    "latency_class": "low",
    "bandwidth_class": "medium",
    "reliability_class": "high",
    "priority": "high",
    "resource_vector": [50, 15, 0.95, 0.9, 0.8]
  }
}
```

#### 数据集2: 卫星网络仿真轨迹数据 (用于C2验证)

| 数据项 | 说明 |
|--------|------|
| **时长** | 24小时仿真 |
| **用户数** | 100-1000 用户 |
| **业务请求** | 100,000+ 请求 |
| **记录内容** | 用户位置、卫星可见性、CSI、业务类型、资源分配、QoS指标 |
| **场景** | 低负载、中负载、高负载、热点区域、切换频繁 |

### 2.3 评估指标

#### 主指标

| 指标 | 公式 | 说明 |
|------|------|------|
| **系统吞吐量** | Σ throughput_i / T | 单位时间内成功传输的数据量 |
| **时延满足率** | N_satisfied / N_total | 时延低于阈值的请求比例 |
| **资源利用率** | Σ allocated / Σ available | 已分配资源占总资源比例 |
| **语义理解准确率** | N_correct / N_total | 正确预测资源需求的样本比例 |

#### 次指标

| 指标 | 说明 |
|------|------|
| 切换成功率 | 无缝切换比例 |
| 公平性指数 (Jain's Fairness) | 资源分配公平程度 |
| 端到端推理时延 | 从输入到输出的总时间 |
| 能效 | 单位能耗的吞吐量 |

### 2.4 实现细节

| 组件 | 配置 |
|------|------|
| **LLM** | Qwen-7B (本地部署) / GPT-3.5-turbo (API) |
| **深度学习框架** | PyTorch 2.0 |
| **RL框架** | Stable-Baselines3 (PPO) |
| **硬件** | NVIDIA RTX 4090 × 2 或 A100 × 1 |
| **随机种子** | 42, 123, 456, 789, 1024 (5个种子取平均) |

## 三、Baseline 对比设计

### 3.1 Baseline 列表

| ID | 方法 | 类型 | 选择理由 | 对比重点 |
|----|------|------|---------|---------|
| **B1** | Random Allocation | 简单基线 | 下界参考 | 展示智能分配的必要性 |
| **B2** | Round Robin | 简单基线 | 公平分配参考 | 展示语义感知的优势 |
| **B3** | Max-SINR | 经典方法 | 工程常用 | 展示语义理解的价值 |
| **B4** | Proportional Fair | 经典方法 | 频域标准 | 对比公平性 |
| **B5** | DQN-based RA | SOTA | 深度RL方法 | 对比决策策略 |
| **B6** | PPO-based RA | SOTA | 当前最优RL | 主要对比对象 |
| **B7** | LLM-SRAF w/o Semantic | 消融变体 | 移除语义模块 | 验证语义贡献 |
| **B8** | LLM-SRAF w/o Fusion | 消融变体 | 移除融合层 | 验证融合贡献 |

### 3.2 公平对比措施

| 措施 | 说明 |
|------|------|
| **统一环境** | 所有方法在相同仿真环境下测试 |
| **相同训练数据** | RL方法使用相同的训练轨迹 |
| **调参公平** | 每个方法进行网格搜索调参，报告最优结果 |
| **相同评估指标** | 使用统一的指标计算代码 |
| **开源实现** | 优先使用官方实现，否则自行复现并开源 |

### 3.3 Baseline 实现来源

| Baseline | 来源 | 状态 |
|----------|------|------|
| B1, B2 | 自行实现 | - |
| B3, B4 | NS-3内置 | - |
| B5 | [参考论文](https://arxiv.org/xxx) | 需复现 |
| B6 | [参考论文](https://arxiv.org/2303.14351) | 需复现 |

## 四、消融实验设计

### 4.1 消融矩阵

| 消融ID | 移除/替换组件 | 预期效果 | 验证目标 |
|--------|-------------|----------|---------|
| **A1** | 移除LLM语义编码器 | ↓ 准确率 15-20% | LLM语义理解的必要性 |
| **A2** | LLM替换为规则匹配 | ↓ 准确率 10-15% | LLM vs 传统方法 |
| **A3** | 移除跨模态融合层 | ↓ 性能 8-12% | 多模态融合的必要性 |
| **A4** | 融合层替换为简单拼接 | ↓ 性能 3-5% | 门控融合的优越性 |
| **A5** | 移除网络状态信息 | ↓ 性能 10-15% | 网络状态的重要性 |
| **A6** | 移除卫星轨道信息 | ↓ 性能 5-8% | 轨道信息的价值 |
| **A7** | PPO替换为DQN | ↓ 性能 3-8% | PPO的选择合理性 |
| **A8** | 无预训练直接RL训练 | ↓ 收敛速度 | 预训练的必要性 |

### 4.2 LLM 对比实验

| LLM选项 | 参数量 | 预期效果 | 对比目标 |
|---------|--------|----------|---------|
| Qwen-7B | 7B | 基线 | 主要使用 |
| Qwen-14B | 14B | ↑ 准确率 2-3%, ↓ 速度 30% | 准确率-速度权衡 |
| GPT-3.5-turbo | - | ↑ 准确率 3-5%, ↑ 成本 | 商业LLM对比 |
| BERT-base | 110M | ↓ 准确率 8-12% | 专用vs通用模型 |
| Rule-based | - | ↓ 准确率 15-20% | 传统方法对比 |

### 4.3 融合策略对比

| 策略 | 描述 | 复杂度 | 预期效果 |
|------|------|--------|---------|
| Early Fusion | 特征级拼接 | O(1) | 基线 |
| Late Fusion | 决策级融合 | O(1) | 可比 |
| Attention Fusion | 跨模态注意力 | O(n²) | ↑2-3% |
| **Gated Fusion** | 门控机制 | O(n) | ↑3-5% ✅ |

## 八、Devil's Advocate 审查

### 8.1 审查清单

| 问题 | 级别 | 回应 |
|------|------|------|
| 语义理解准确率能否真实反映资源分配效果? | MAJOR | 需增加端到端性能验证 |
| 仿真环境与真实卫星网络的差异? | MAJOR | 在论文中明确说明局限性 |
| LLM API成本是否影响实用性? | MINOR | 提供本地部署方案 |
| 是否遗漏重要Baseline? | MINOR | 已涵盖主要方法类型 |
| 数据泄露风险? | CRITICAL | ✅ 已检查：训练/测试严格分离 |

### 8.2 潜在问题与应对

| 问题 | 风险 | 应对策略 |
|------|------|---------|
| LLM推理延迟过高 | 高 | 预计算常见语义向量 + 缓存 |
| RL训练不收敛 | 中 | 使用预训练初始化 + 课程学习 |
| 数据不足 | 中 | 数据增强 + 合成数据 |
| Baseline复现困难 | 低 | 联系原作者 / 使用近似实现 |
