# SatFault-LLM: 卫星网络故障根因智能定位

**项目状态**: 🚀 启动中
**研究方向**: 卫星网络 + 大语言模型
**研究类型**: 负面结果转化后的新方向

---

## 核心问题

> 如何利用大语言模型(LLM)对卫星网络中多源异构日志进行语义理解与关联分析，实现故障根因的智能定位？

---

## 为什么选择这个方向？

### 与之前失败案例的对比

| 维度 | LLM-SRAF (失败) | SatFault-LLM (本方案) |
|------|-----------------|----------------------|
| 问题类型 | 结构化决策 | 非结构化文本理解 |
| 信息来源 | 优先级参数（已结构化） | 多源异构日志（非结构化） |
| LLM必要性 | ❌ 冗余 | ✅ 必需 |
| 创新空间 | 有限 | 高 |

### LLM必要性检验

| 检验项 | 结果 |
|--------|------|
| 信息来源 | ✅ 故障日志是非结构化文本 |
| 任务本质 | ✅ 需要语义理解+因果推理 |
| 替代可能性 | ✅ 结构化特征无法替代 |

---

## 三大创新点

1. **异构日志语义统一**
   - LLM将多源异构日志映射到统一语义空间
   - 无需手工特征工程

2. **跨系统因果推理**
   - GNN建模日志间因果关系
   - 发现故障传播链

3. **可解释根因定位**
   - 生成自然语言解释
   - 提升运维可信度
